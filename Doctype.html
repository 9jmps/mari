Documentos html

HTML es un lenguaje de marcado que se utiliza para el desarrollo de paginas de internet. Los documentos en HTML proporcionan una estructura al contenido que aparece en un sitio web, como imágenes, texto o videos. HTML significa lenguaje de marcado de hipertexto que es:

  • Un lenguaje de marcadoes un lenguaje de computadora que define la estructura y la presentación del texto en bruto.
  • En HTML, la computadora puede interpretar el texto sin procesar que está envuelto en elementos HTML.
  • El hipertexto es un texto que se muestra en una computadora o dispositivo que brinda acceso a otro texto a través de enlaces, también conocidos como hipervínculos .

Declaracion del tipo de documentos

Cuando escribimos nuestro documento HTML, lo primero que tenemos que escribir es el Doctype. El Doctype es la declaracion del tipo de documento, nos sirve para indicar que nuestro documento esta escrito siguiendo la estrucruta determinada para un DTD concreto.

La declaracion siempre debe ser la primera línea de código en sus archivos HTML. Esto le permite al navegador saber qué versión de HTML puede esperar.

Etiquetas en HTML

Html esta compuesto de elementos, un ejemplo de lo que pueden definir seria los parrafos, titulos, listas etc. Estos elementos estructuran la pagina web y defienden su contenido. Una declaracion en html esta compuesta por una etiqueta de apertura y una etiqueta de cierre:


  • Elemento HTML (o simplemente, elemento): una unidad de contenido en un documento HTML formado por etiquetas HTML y el texto o medio que contiene.
  • Etiqueta HTML: el nombre del elemento, rodeado por un corchete angular de apertura y cierre.
  • Etiqueta de apertura: la primera etiqueta HTML utilizada para iniciar un elemento HTML. El tipo de etiqueta está rodeado de paréntesis de ángulo de apertura y cierre.
  • Contenido: la información (texto u otros elementos) contenida entre las etiquetas de apertura y cierre de un elemento HTML.
  • Etiqueta de cierre: la segunda etiqueta HTML utilizada para finalizar un elemento HTML. Las etiquetas de cierre tienen una barra inclinada ( /) dentro de ellas, justo después del soporte del ángulo izquierdo.

Estructura Html

HTML se organiza como una colección de relaciones de árbol genealógico. Cuando un elemento está contenido dentro de otro elemento, se considera el elemento secundario de ese elemento. Se dice que el elemento hijo está anidadodentro del elemento padre.

En el ejemplo anterior, el pelemento está anidado dentro del bodyelemento. El pelemento se considera un elemento secundario del bodyelemento y el bodyelemento se considera el elemento primario. También puede ver que hemos agregado dos espacios de sangría (usando la barra de spacio) para una mejor legibilidad. Dado que puede haber múltiples niveles de anidación, esta analogía puede extenderse a los nietos, bisnietos y más. La relación entre los elementos y sus elementos ancestros y descendientes se conoce como jerarquía .
En este ejemplo, el body elemento es el padre del divelemento. Tanto el h1 y plos elementos son hijos del divelemento. Debido a que los elementos h1 y p están en el mismo nivel, se consideran hermanos y son nietos del elemento body.

Divs

Uno de los elementos más populares en HTML es el elemento div.Es la abreviatura de "división" o un contenedor que divide la página en secciones. Estas secciones son muy útiles para agrupar elementos HTML juntos, puede contener cualquier texto u otro elemento HTML, como enlaces, imágenes o videos. Recuerde siempre agregar dos espacios de sangría cuando anide elementos dentro de divs para una mejor legibilidad.

Comentarios en HTML

Los archivos HTML también le permiten agregar comentarios a su código.

Los caracteres intermedios serán ignorados por su navegador. Incluir comentarios en su código es útil por muchas razones:
  • Le ayudan a usted (y a otros) a entender su código si decide regresar y revisarlo en una fecha mucho más posterior.
  • Le permiten experimentar con un nuevo código, sin tener que eliminar el código antiguo.

Espacios e identacion

Espacio en blanco

El resto de esta lección se centrará en algunas herramientas que usan los desarrolladores para hacer que el código sea más fácil de interpretar. A medida que crece el código en un archivo HTML, cada vez es más difícil hacer un seguimiento de cómo se relacionan los elementos. Los programadores utilizan dos herramientas para visualizar la relación entre los elementos: espacios en blanco y sangría .
El navegador ignora los espacios en blanco en los archivos HTML cuando representa una página web, por lo que se puede utilizar como una herramienta para facilitar la lectura y el seguimiento del código.

Sangria

La segunda herramienta que usan los desarrolladores web para facilitar la lectura de la estructura del código es la sangría. Los espacios se insertan utilizando las barras de espacio y tab en su teclado.

El World Wide Web Consortium , o W3C, es responsable de mantener los estándares de estilo de HTML. Al momento de escribir, el W3C recomienda 2 espacios de sangría al escribir el código HTML. Aunque su código funcionará sin exactamente dos espacios, este estándar es seguido por la mayoría de los desarrolladores web profesionales. La sangría se utiliza para visualizar fácilmente qué elementos están anidados dentro de otros elementos.


En el ejemplo anterior, Paragraph 1y la div etiqueta están anidadas dentro de la body etiqueta, por lo que están sangradas dos espacios. El Paragraph 2elemento está anidado dentro de la divetiqueta, por lo que se sangra dos espacios adicionales.
